Ünite 4 — İfadeler ve Deyimler (Expressions and statements)

Bu ünite, “kodun ne zaman neyi hesapladığı” ile ilgilidir. Sınavda en çok şu tip sorular gelir:

  • İşlem önceliği (precedence) / ilişkililik (associativity)
  • Yan etki (side effect) ve değerlendirme sırası (evaluation order)
  • Pointer ile yapılan güncellemenin ifadeye etkisi (çıktı soruları)

4.1 Expressions — İfadeler

İfade: bir değer üreten parça. Örn: a + b, f(x), *p.

İşlem önceliği kısa özeti (C)

  • Parantez () her şeyi kontrol eder.
  • Çarpma/bölme * / % toplama/çıkarma + -’dan önce gelir.
  • Atama = en sonda çalışır.

Yan etki (side effect)

i++, *p = 5 gibi işlemler programın durumunu değiştirir. Yan etki varsa, ifadeyi okurken “bu ifade sadece değer mi üretiyor, yoksa değiştiriyor mu?” diye sor.

4.2 Statements — Deyimler

Deyim: genelde program akışını kontrol eder. Örn: if, while, return, break.

ÇIKTI SORUSU MANTIĞI: Pointer + değerlendirme sırası

Senin çıktısı istenen sorunda kilit nokta şu:

  1. fun(&i) çağrısı i’yi değiştiriyor (pointer ile).
  2. Aynı değişken, aynı satırda hem i/2 hem fun(&i) içinde kullanılıyor.
  3. Bu iki parçanın hangi sırayla hesaplanacağı sonucu etkiler.
Önemli: C standartlarında bazı ifadelerde değerlendirme sırası “garanti edilmez”. Bu yüzden sınavlar genelde ya açık sıraya sahip örnek verir ya da “derleyici şu sırayla hesaplıyor varsay” gibi düşünür. En güvenli yaklaşım: ifadeyi adım adım parçala.

Verilen örnek (C)

int fun(int *k) {
    *k += 4;
    return 3 * (*k) - 1;
}

int main() {
    int i = 10, j = 10, sum1, sum2;

    sum1 = (i / 2) + fun(&i);
    sum2 = fun(&j) + (j / 2);

    printf("sum1: %d\t sum2: %d\n", sum1, sum2);
}

Adım adım hesap (güvenli yöntem)

  • Başta i=10. fun(&i) çağrılınca: i = 10 + 4 = 14, dönüş: 3*14 - 1 = 41.
  • i/2 ifadesi, çağrıdan önce hesaplanırsa: 10/2 = 5 → sum1 = 5 + 41 = 46.
  • j=10. fun(&j) → j=14, dönüş 41.
  • j/2 çağrıdan sonra hesaplanırsa: 14/2 = 7 → sum2 = 41 + 7 = 48.

Neden Python ekledim?

Python burada “çıktı sorusunu çözmek için şart” değil; ama aynı mantığı daha görünür yapmak için iyi: fonksiyonun yan etkisini ve sıraya göre değişimi daha rahat gözleyebilirsin.

def fun_box(box):
    box[0] += 4
    return 3 * box[0] - 1

i = [10]
sum1 = (i[0] // 2) + fun_box(i)  # önce i//2, sonra fun (soldan sağa)
print("sum1:", sum1, "i:", i[0])
Python’da ifadeler soldan sağa değerlendirilir; bu yüzden çıktıyı daha deterministik görürsün. C’de ise bu tarz sorularda “ifadeyi parçala” tek güvenli yoldur.
