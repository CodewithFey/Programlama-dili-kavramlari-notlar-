Ünite 11: Eşzamanlılık (Concurrency) ve Ağ Temelleri

1) Eşzamanlılık nedir, paralellik nedir?

  • Concurrency (eşzamanlılık): Birden fazla işin ilerlemesi (zaman dilimlenebilir). Tek çekirdekte bile olabilir.
  • Parallelism (paralellik): İşlerin gerçekten aynı anda çalışması (genellikle çok çekirdek).

Neden önemli? Paylaşılan veri olduğunda “yarış durumu (race condition)” ve “kilitlenme (deadlock)” gibi problemler çıkar.

2) Race condition örneği (C / pthread)

Aşağıdaki kodda iki thread aynı değişkeni kilitsiz arttırırsa sonuç tahmin edilenden küçük çıkabilir.

#include <pthread.h>
#include <stdio.h>

long counter = 0;

void* inc(void* arg){
  for(long i=0;i<1000000;i++) counter++; // kritik bölge (kilitsiz)
  return NULL;
}

int main(){
  pthread_t t1, t2;
  pthread_create(&t1, NULL, inc, NULL);
  pthread_create(&t2, NULL, inc, NULL);
  pthread_join(t1, NULL);
  pthread_join(t2, NULL);
  printf("counter=%ld\n", counter); // 2000000 beklenir ama bazen daha az gelir
}

Neden? counter++ tek adım değildir (load‑inc‑store). Araya girme olunca kayıp güncelleme olur.

3) Mutex ile çözüm (kilitleme)

#include <pthread.h>

long counter = 0;
pthread_mutex_t m = PTHREAD_MUTEX_INITIALIZER;

void* inc(void* arg){
  for(long i=0;i<1000000;i++){
    pthread_mutex_lock(&m);
    counter++;
    pthread_mutex_unlock(&m);
  }
  return NULL;
}

Trade‑off: Doğruluk ↑ ama kilit maliyeti yüzünden hız düşebilir.

4) Python ile aynı fikir (thread + lock)

import threading

counter = 0
lock = threading.Lock()

def inc():
    global counter
    for _ in range(1_000_000):
        with lock:
            counter += 1

t1 = threading.Thread(target=inc)
t2 = threading.Thread(target=inc)
t1.start(); t2.start(); t1.join(); t2.join()
print(counter)

Python örneği “kilit fikrini” göstermek içindir. (GIL ayrıntısı bu dersin kapsamına göre değişebilir.)

5) Ağ (network) temel kavramlar

  • Socket: İki uç arasında veri alışverişi için uç nokta (endpoint).
  • TCP: Bağlantı‑yönelimli, sıralı ve güvenilir aktarım.
  • UDP: Bağlantısız, hızlı ama kayıp/yeniden sıralama olabilir.
  • Client/Server: İstemci istek atar, sunucu dinler ve cevaplar.
