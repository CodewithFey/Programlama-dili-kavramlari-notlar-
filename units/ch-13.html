Ünite 13: Derleme, Bağlama, Kütüphaneler

Bu ünite, kaynak kodun çalışır programa dönüşme sürecini anlatır: ön işlem → derleme → bağlama → çalıştırma. Sınavda özellikle ayrı derleme ve statik/dinamik kütüphane sorulur.

Altın cümle: C ve C++ “ayrı derleme + bağlama (linking)” modelini kullanır.

13.1 Derleme aşamaları (C/C++)

  1. Preprocessor: #include, #define
  2. Compiler: C → assembly
  3. Assembler: assembly → .o
  4. Linker: .o + kütüphaneler → executable

13.2 Ayrı derleme örneği (3 dosya)

fonksiyon.h

#ifndef FONKSIYON_H
#define FONKSIYON_H

float alan(float g, float y);
float cevre(float g, float y);

#endif

fonksiyon.c

#include "fonksiyon.h"

float alan(float g, float y){ return g * y; }
float cevre(float g, float y){ return 2 * (g + y); }

main.c

#include <stdio.h>
#include "fonksiyon.h"

int main(){
  printf("Alan: %.2f\n", alan(5,3));
  printf("Çevre: %.2f\n", cevre(5,3));
}

Derleme komutu

gcc -c fonksiyon.c -o fonksiyon.o
gcc -c main.c -o main.o
gcc main.o fonksiyon.o -o dikdortgen

Pratikte çoğu zaman tek satırda da yazılır:

gcc main.c fonksiyon.c -o dikdortgen

13.3 Statik vs Dinamik kütüphane

Özellik Statik (.a) Dinamik (.so / .dll)
Bağlama Derleme/Link zamanı Çalışma zamanı
Boyut Daha büyük executable Daha küçük executable
Paylaşım Paylaşılamaz Paylaşılıp güncellenebilir

Statik kütüphane oluşturma

gcc -c fonksiyon.c -o fonksiyon.o
ar rcs libfonksiyon.a fonksiyon.o
gcc main.c -L. -lfonksiyon -o program

Dinamik kütüphane oluşturma

gcc -fPIC -c fonksiyon.c -o fonksiyon.o
gcc -shared -o libfonksiyon.so fonksiyon.o
gcc main.c -L. -lfonksiyon -o program

13.4 Sık yapılan hatalar

  • Header guard kullanmamak → tekrar tanım hatası
  • -L. ve -l kullanımını karıştırmak
  • Dinamik kütüphanede çalışma zamanı arama yolu (LD_LIBRARY_PATH) unutmak

13.5 Çıkabilecek soru + cevap

  1. Ayrı derleme nedir?
    Modüllerin ayrı derlenip linker ile birleştirilmesidir.
  2. C dili hangi derleme grubundadır?
    Ayrı derleme.
  3. Statik ve dinamik kütüphane farkı?
    Statik programa gömülür, dinamik çalışmada yüklenir/paylaşılır.

13.6 Mini Test

  1. ar rcs hangi amaçla kullanılır?
  2. Dinamik kütüphane neden “paylaşılabilir”dir?

Cevap: (1) .a oluşturmak (2) birden çok program aynı .so’yu kullanabilir
