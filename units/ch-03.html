Ünite 3 — Tipler, Değerler ve Bildirimler (Types, values and declarations)

Bu ünite sınavın en “puan getiren” kısmıdır çünkü birçok soru buradan türetilir: value vs reference, binding, pointer, enum, boolean, vb.

Altın cümle: Tip “hangi değerler kümesi + hangi işlemler” demektir. Bildirim (declaration) ise “isim → tipe bağlama” işlemidir.

3.1 Names — İsimler

  • Identifier: değişken/fonksiyon/sınıf adı.
  • Scope: ismin görülebildiği bölge (Ünite 5/6 ile de bağlanır).
  • Lifetime: ismin işaret ettiği saklamanın yaşadığı süre.

3.2 Declarations and binding — Bildirim ve bağlama

Binding = “bir şeyin bir şeye bağlanması”. Örn: isim → adres, isim → tip, çağrı → fonksiyon gövdesi.

  • Statik (compile-time) bağlama: derleyici karar verir. (C’de birçok şey)
  • Dinamik (run-time) bağlama: çalışma anında karar verilir. (OOP'de sanal metotlar, dinamik tipli diller)

Örnek: C’de tip bağlama statiktir

// a'nın tipi derleme zamanında bellidir: int
int a = 10;
// a = 3.14;  // HATA: double -> int uyumsuzluğu (derleyici yakalar)

3.3 Type definitions — Tip tanımları

Yeni tip oluşturma 2 amaçla kullanılır:

  • Okunabilirlik: kod “alan” hakkında konuşur (metre, TL, öğrenciNo).
  • Soyutlama: ayrıntıyı saklayıp arayüz sunma (Ünite 2.9).
// typedef ile "yeni isim" veriyoruz (C)
typedef unsigned long ulong;
ulong x = 42;

3.4 Numeric data types — Sayısal tipler

  • İnt: tam sayılar, taşma (overflow) olabilir.
  • Float/Double: kayan nokta, yuvarlama hatası vardır (0.1 + 0.2 problemi).
// Kayan nokta hassasiyeti örneği
#include <stdio.h>

int main() {
    double x = 0.1 + 0.2;
    printf("%.17f\n", x); // 0.30000000000000004 görebilirsin
    return 0;
}
x = 0.1 + 0.2
print(f"{x:.17f}")

3.5 Logical types — Mantıksal tipler (boolean)

  • C’de stdbool.h ile bool kullanılır.
  • C’de 0 = false, 0 dışı = true geleneği vardır.
  • Python’da True/False ayrı tiptir; ama 0 “falsey” sayılır.

3.6 Character types — Karakter tipleri

  • C’de char aslında küçük bir tam sayıdır (ASCII/byte).
  • Unicode konusu C’de ayrı kütüphanelerle yönetilir.
  • Python’da str Unicode’dur, tek karakter de bir str'dir.

3.7 Enumeration types — Enum

enum Gun { PAZARTESI, SALI, CARSAMBA };
enum Gun g = SALI;

Sınav ipucu: Enum okunabilirlik sağlar ama C’de tip güvenliği sınırlıdır (int ile karışabilir).

3.8 Reference and pointer variables — Referans ve pointer

Bu başlık doğrudan “büyüklük (value) – referans (reference) tipleri” sorularına bağlanır.

  • Value: değişken değeri tutar. Kopyalama “değeri” kopyalar.
  • Reference/Pointer: değişken bir adresi tutar. Kopyalama “adresi” kopyalar.

C’de pointer ile “referans benzeri” davranış

#include <stdio.h>

void arttir(int *p) { // p: adres taşıyor
    (*p)++;           // adresin gösterdiği yeri değiştiriyoruz
}

int main() {
    int a = 10;
    arttir(&a);
    printf("%d\n", a); // 11
    return 0;
}

C++’ta gerçek referans

#include <iostream>
using namespace std;

void arttir(int &x){ // x, a'nın takma adıdır (alias)
    x++;
}

int main(){
    int a = 10;
    arttir(a);
    cout << a; // 11
}

Python’da “isim bağlama” (sınavda çok karıştırılır)

a = [10, 20, 30]
b = a       # b aynı listeyi işaret eder
b[0] = 1
print(a)    # [1, 20, 30]  (aynı obje)
Not: Python “call by object reference / sharing” diye geçer. Yani isimler objelere bağlanır; değiştirilebilir (mutable) objelerde etkisi “referans gibi” görünür.
