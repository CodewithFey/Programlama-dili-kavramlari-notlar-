Ünite 2 — Tarihsel İnceleme (Historical survey)

Bu ünite, programlama dillerinin “neden böyle” olduğunu anlamak için tarihsel bir çerçeve kurar. Sınavda bu bölüm genelde paradigmaların doğuşu, dil özelliklerinin motivasyonu ve “hangi problem hangi dil özelliğini doğurdu?” şeklinde sorulur.

Akılda kalsın: Donanım (makine) kısıtları → dil tasarımı → yazılım mühendisliği ihtiyaçları. Tarih boyunca diller, o dönemin baskın problemine “pratik” bir çözüm olarak doğdu.

2.1 Early machines — İlk makineler

  • Durum: İlk dönemlerde programlar makine kodu/assembly seviyesinde yazılıyordu.
  • Problemler: taşınabilirlik yok, bakım zor, hata ayıklama çok maliyetli.
  • Sonuç: “insana daha yakın” notasyon ihtiyacı → yüksek seviyeli diller.

2.2 Fortran — Sayısal hesaplama devrimi

Fortran’ın temel motivasyonu: mühendislik/ bilimsel hesaplamayı daha hızlı geliştirmek. Burada iki büyük fikir çıkar:

  • Derleyici optimizasyonu: Yüksek seviyeli kodun hızlı makine koduna dönüşmesi.
  • Döngüler / diziler: Sayısal hesaplamanın bel kemiği.

Mini örnek: Döngü fikri (C)

// Döngüler: sayısal işlerin omurgası
#include <stdio.h>

int main() {
    double sum = 0.0;
    for (int i = 1; i <= 100; i++) {
        sum += 1.0 / i;
    }
    printf("sum=%.6f\n", sum);
    return 0;
}

Aynı fikir (Python)

sum_ = 0.0
for i in range(1, 101):
    sum_ += 1.0 / i
print(f"sum={sum_:.6f}")

2.3 Algol — Yapısallık, bloklar, kapsam

Algol ailesi (Algol60 vb.) ile çok kritik kavramlar popülerleşti:

  • Blok yapısı (begin/end benzeri) ve lexical/static scope
  • BNF ile sözdizimi tanımlama kültürü
  • Algoritma anlatımı için daha temiz yapı

Bu kısım sınavda doğrudan “statik kapsam nedir?” gibi sorulara bağlanır.

2.4 Business data processing languages — İş verisi odaklı diller

İş dünyası, sayısal hesaplardan farklı olarak kayıt/fiş/fatura gibi veri yoğun işleri çözüyor: bu yüzden kayıt (record), dosya, raporlama gibi özellikler öne çıktı.

2.5 General or multipurpose languages — Genel amaçlı diller

Tek bir alana değil, birçok alana hizmet eden dillerin hedefi: taşınabilirlik + üretkenlik + makul performans. C/C++ bu çizgide çok güçlü örneklerdir.

2.6 Developing programs interactively — Etkileşimli geliştirme

Bu başlık, “edit-compile-run” döngüsünün ağır olduğu dönemlere tepki olarak gelişen yaklaşımları kapsar: REPL, yorumlayıcılar, hızlı prototipleme.

  • Python burada klasik örnek: hızlı dene–gör.
  • C daha çok derleme ve bağlama adımlarıyla ilerler.

2.7 Special-purpose languages — Özel amaçlı diller

Belirli problem sınıfları için “en iyi” ifade gücü: SQL (sorgu), Prolog (mantık), MATLAB (matris), Regex (desen).

2.8 Systems programming languages — Sistem programlama dilleri

İşletim sistemi, sürücü, derleyici gibi düşük seviye yazılımlar için:

  • Doğrudan bellek kontrolü (pointer)
  • Deterministik performans
  • Düşük seviye erişim

Bu yüzden C, “sistem programlama” denince en güçlü tarihsel örnektir.

2.9 Modules, classes and abstract data types — Modüller / sınıflar / ADT

Zamanla programlar büyüdükçe modülerlik zorunlu oldu.

  • Modül: birim bir dosya/ paket gibi düşün.
  • ADT (Soyut Veri Tipi): “ne yapar?”ı dışarı aç, “nasıl yapar?”ı gizle.
  • Sınıf: veri + davranışı aynı yapıda toplar (OOP).

ADT fikri (C'de header + .c)

// stack.h — dışarı görünen arayüz (NE yapıyor?)
#ifndef STACK_H
#define STACK_H

void push(int x);
int pop(void);

#endif
// stack.c — içerideki uygulama (NASIL yapıyor?)
#include "stack.h"
#define N 100
static int a[N];
static int top = 0;

void push(int x){ a[top++] = x; }
int pop(void){ return a[--top]; }

2.10 Functional and logic languages — Fonksiyonel ve mantıksal diller

İki büyük yaklaşım:

  • Fonksiyonel: yan etkiler azaltılır, fonksiyonlar birinci sınıftır (map/filter).
  • Mantıksal (Prolog): “nasıl” yerine “ne”yi yazarsın; çıkarım yapar.

Prolog mini örnek

% Fact
insan(selim).

% Rule
olumlu(X) :- insan(X).

% Query
% ?- olumlu(selim).

2.11 Conclusions — Sonuç

  • Yeni özellikler, genelde eski dillerin “eksiklerine” yanıt olarak gelir.
  • Paradigma seçimi: problem tipine göre avantaj/dezavantaj taşır.
