Ünite 9: Fonksiyonel Diller

1) Fonksiyonel programlama ne demek?

Merkez fikir: Programı, “durumu adım adım değiştiren komutlar” yerine fonksiyonların bileşimi (composition) olarak düşünürsün.

  • Saf (pure) fonksiyon: Aynı girdi → her zaman aynı çıktı. Yan etki (I/O, global değişken değiştirme) yoktur.
  • İmmutability: Veri mümkün olduğunca değişmez kabul edilir; yeni değer üretirsin.
  • Referential transparency: Bir ifadeyi sonucu ile değiştirsen programın anlamı bozulmaz.

Neden sınav sorulur? “Yan etki – durum – test edilebilirlik – paralellik” ilişkisini ölçer.

2) Higher‑order function (yüksek mertebeden fonksiyon)

Fonksiyonların parametre olarak alınabildiği / geri döndürülebildiği durum.

# Python: fonksiyon parametre (map/filter) örneği
nums = [1, 2, 3, 4, 5, 6]
evens = list(filter(lambda x: x % 2 == 0, nums))
squares = list(map(lambda x: x*x, evens))
print(squares)  # [4, 16, 36]

C'de karşılığı: Fonksiyon pointer'ı ile benzer mantık kurulabilir.

#include <stdio.h>

int is_even(int x){ return x % 2 == 0; }

// Basit filter: koşulu sağlayanları out'a kopyalar
int filter(const int* in, int n, int* out, int (*pred)(int)){
  int k=0;
  for(int i=0;i<n;i++) if(pred(in[i])) out[k++] = in[i];
  return k; // kaç eleman yazdık
}

int main(){
  int a[] = {1,2,3,4,5,6};
  int out[6];
  int m = filter(a, 6, out, is_even);
  for(int i=0;i<m;i++) printf("%d ", out[i]);
}

3) Sık kavramlar

  • Map: her elemana fonksiyon uygula.
  • Filter: koşulu sağlayanları seç.
  • Reduce/Fold: listeyi tek değere indir (toplam, çarpım, maksimum...).
  • Recursion: döngü yerine özyineleme.

Sık yapılan hata

Fonksiyonel yaklaşımı “Python'da lambda yazmak” sanmak. Asıl konu; yan etkileri azaltmak ve hesabı ifade olarak kurmak.
