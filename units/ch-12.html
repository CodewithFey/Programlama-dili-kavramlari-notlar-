Ünite 12: Exception Handling

Bu ünite, program çalışırken oluşan hataların kontrollü biçimde yakalanması (handle edilmesi) konusunu anlatır. Sınavda genelde iki tarz soru gelir: (1) try/catch mantığını anlat (2) C’de neden zordur?

Altın cümle: Exception handling, “hata olunca programı çökertmek yerine, hatayı yakalayıp güvenli bir yola sapmak”tır.

12.1 Neden gerekir?

  • Dosya bulunamadı (FileNotFound)
  • Sıfıra bölme
  • Ağ bağlantısı koptu
  • Bellek yetersiz

12.2 C# ile try / catch / finally

class Program
{
  static void Main()
  {
    try
    {
      Console.Write("a: ");
      int a = int.Parse(Console.ReadLine());

      Console.Write("b: ");
      int b = int.Parse(Console.ReadLine());

      // Hata: b=0 ise DivideByZeroException
      int c = a / b;
      Console.WriteLine("Sonuç: " + c);
    }
    catch (DivideByZeroException)
    {
      Console.WriteLine("Sıfıra bölme hatası!");
    }
    catch (FormatException)
    {
      Console.WriteLine("Sayı girişi bekleniyordu.");
    }
    finally
    {
      // Hata olsa da olmasa da çalışır: kaynak kapatma, log, vb.
      Console.WriteLine("Program bitti.");
    }
  }
}

finally özellikle “dosya açtım → mutlaka kapatmalıyım” gibi durumlarda kritiktir.

12.3 Python ile try / except / finally

try:
    a = int(input("a: "))
    b = int(input("b: "))
    print("Sonuç:", a // b)
except ZeroDivisionError:
    print("Sıfıra bölme hatası!")
except ValueError:
    print("Sayı girişi bekleniyordu.")
finally:
    print("Program bitti.")

12.4 C’de exception yok: nasıl yönetilir?

C’de try/catch yoktur. Bu yüzden hatalar genelde şu yollarla yönetilir:

  • Dönüş değeri ile hata kodu dönmek
  • errno ile hata bilgisini okumak
  • Nadiren: setjmp/longjmp ile kontrol sıçratmak (riskli)

(a) Dönüş değeriyle hata

#include <stdio.h>

int bol(int a, int b, int *sonuc){
  if(b == 0) return 0;     // 0 = hata
  *sonuc = a / b;
  return 1;                // 1 = başarılı
}

int main(){
  int out;
  if(!bol(10, 0, &out)){
    printf("Hata: b=0\n");
  } else {
    printf("Sonuç=%d\n", out);
  }
}

(b) errno örneği

#include <stdio.h>
#include <errno.h>
#include <stdlib.h>

int main(){
  char *end;
  long x = strtol("12x", &end, 10);
  if(end == NULL || *end != '\0'){
    printf("Hatalı sayı: %s\n", end);
  }
  if(errno == ERANGE){
    printf("Aralık dışı!\n");
  }
}

12.5 Sık yapılan hatalar

  • catch bloklarını “genel”den “özel”e yazmak (C#/Java’da bazı durumlarda engel olur)
  • finally/with kullanmamak → kaynak sızıntısı
  • C’de hata kodu kontrolünü atlamak

12.6 Çıkabilecek soru + cevap

  1. Exception handling nedir?
    Beklenmedik durumları yakalayıp programı güvenli şekilde devam ettirme mekanizmasıdır.
  2. C’de neden zordur?
    Dil düzeyinde try/catch yoktur; dönüş değeri/errno gibi manuel yöntemlerle yapılır.
