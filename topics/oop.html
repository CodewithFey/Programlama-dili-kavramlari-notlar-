OOP Derinlemesine (C++ + Python)

Bu bölüm ünite akışından bağımsız, sınavda çok çıkan OOP kavramlarını çok detaylı toparlar. Her başlıkta: tanım → mantık → C++ örnek → Python örnek → sık hata.

1) Encapsulation (Kapsülleme)

Tanım: Veriyi (state) ve o veri üzerinde çalışan fonksiyonları (behavior) bir araya getirmek ve dışarıya kontrollü erişim vermektir. Amaç: “yanlış kullanımı” azaltmak ve sistemi modüler tutmaktır.

C++ örneği

#include <iostream>
using namespace std;

class Account {
private:
  double balance;              // dışarıdan direkt erişim yok
public:
  Account(double b): balance(b) {}
  void deposit(double x){ if(x > 0) balance += x; }
  bool withdraw(double x){
    if(x <= balance){ balance -= x; return true; }
    return false;
  }
  double getBalance() const { return balance; }
};

int main(){
  Account a(100);
  a.deposit(50);
  cout << a.getBalance() << endl;
}

Python örneği

class Account:
    def __init__(self, balance: float):
        self._balance = balance   # konvansiyonel "private"

    def deposit(self, x: float):
        if x > 0:
            self._balance += x

    def withdraw(self, x: float) -> bool:
        if x <= self._balance:
            self._balance -= x
            return True
        return False

    @property
    def balance(self):
        return self._balance

Sık hata: Her şeyi public bırakmak → sınıf “data bag” olur, kurallar dağılır.

2) Inheritance (Kalıtım)

Tanım: Alt sınıfın üst sınıfın özellik ve davranışlarını devralmasıdır. “is-a” ilişkisidir.

C++

#include <iostream>
using namespace std;

class Animal{
public:
  virtual void speak(){ cout << "..." << endl; }
  virtual ~Animal() = default;
};

class Dog : public Animal{
public:
  void speak() override { cout << "hav hav" << endl; }
};

Python

class Animal:
    def speak(self):
        return "..."

class Dog(Animal):
    def speak(self):
        return "hav hav"

Sık hata: “has-a” ilişkisini kalıtımla çözmek (kompozisyon daha doğru olabilir).

3) Polymorphism (Çok Şekillilik) + Dynamic Dispatch

Mantık: Üst sınıf türünden bir referansla, alt sınıfın override ettiği metodu çağırabilmektir.

C++

Animal* a = new Dog();
a->speak(); // virtual sayesinde Dog::speak çalışır
delete a;

Python

def make_speak(animal: Animal):
    print(animal.speak())

make_speak(Dog())  # duck typing ile çalışır

Sınav ipucu: C++'ta polymorphism için virtual şarttır; yoksa “static binding” olur.

4) Overload vs Override

  • Overload: Aynı isim, farklı imza (C++/C#). Python’da gerçek overload yok; varsayılan argüman veya *args kullanılır.
  • Override: Alt sınıf, üst sınıf metodunu yeniden yazar (virtual/override).

C++ overload

int sum(int a,int b){ return a+b; }
double sum(double a,double b){ return a+b; }

Python benzeri

def sum2(a, b):
    return a + b   # dinamik tip: int de olur float da

5) Abstract class / Interface

C++’ta saf sanal metot ile “abstract” yapılır. Python’da ABC ile yapılabilir.

C++

class Shape{
public:
  virtual double area() const = 0; // pure virtual
  virtual ~Shape() = default;
};

Python

from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self) -> float:
        pass

6) Sınav tipi kısa sorular (cevaplı)

  1. Virtual neden gerekli?
    Çalışma zamanında doğru override edilmiş metodu seçmek için (dynamic dispatch).
  2. Encapsulation neyi “gizler”?
    Verinin temsilini ve iç kurallarını; dışarıya sadece kontrollü arayüz verir.
