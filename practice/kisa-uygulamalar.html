Kısa Uygulama Soruları

Bu bölüm, hocanın “kısa uygulama” tarzı sorularına göre hazırlanmıştır. Kodlar C ağırlıklı; gerektiğinde Python eklenmiştir.

Soru 1 — evenNumbers fonksiyonu

İstenen: Bir listede/dizide gezerken çift sayıları ayrı bir listeye ekleyen fonksiyon.

C (dinamik dizi mantığı basit tutuldu)

#include <stdio.h>

int is_even(int x) { return x % 2 == 0; }

int main() {
    int arr[] = {3, 10, 7, 8, 12, 5};
    int n = sizeof(arr) / sizeof(arr[0]);

    int even[100];  // basitlik için sabit kapasite
    int k = 0;

    for (int i = 0; i < n; i++) {
        if (is_even(arr[i])) {
            even[k++] = arr[i];
        }
    }

    printf("Ciftler: ");
    for (int i = 0; i < k; i++) printf("%d ", even[i]);
    printf("\n");
    return 0;
}

Python (liste üretimi)

nums = [3, 10, 7, 8, 12, 5]
even_numbers = [x for x in nums if x % 2 == 0]
print(even_numbers)

Soru 2 — Personel / Muhendis: overload ve override

İstenen: Overload = aynı isim, farklı parametre; Override = alt sınıfın virtual metodu ezmesi.

#include <iostream>
using namespace std;

class Personel {
public:
    // overload: ayni isim, farkli parametre listesi
    double maasHesapla(double taban) { return taban; }
    double maasHesapla(double taban, double prim) { return taban + prim; }

    // override icin virtual
    virtual void yazdir() { cout << "Personel\n"; }
};

class Muhendis : public Personel {
public:
    void yazdir() override { cout << "Muhendis\n"; } // override
};

int main() {
    Personel p;
    cout << p.maasHesapla(30000) << "\n";
    cout << p.maasHesapla(30000, 5000) << "\n";

    Personel *ref = new Muhendis();
    ref->yazdir(); // Muhendis
    delete ref;
}

Python ek (override):

class Personel:
    def yazdir(self):
        print("Personel")

class Muhendis(Personel):
    def yazdir(self):
        print("Muhendis")  # override

Soru 3 — C'de dikdörtgen alan/çevre + gcc komutu

İstenen: Yapısal programlama + ayrı derleme: header, source, main.

// fonksiyon.h
#ifndef FONKSIYON_H
#define FONKSIYON_H
float alan(float g, float y);
float cevre(float g, float y);
#endif
// fonksiyon.c
#include "fonksiyon.h"
float alan(float g, float y)  { return g * y; }
float cevre(float g, float y) { return 2 * (g + y); }
// main.c
#include <stdio.h>
#include "fonksiyon.h"

int main() {
    float g = 5, y = 3;
    printf("Alan: %.2f\n", alan(g,y));
    printf("Cevre: %.2f\n", cevre(g,y));
    return 0;
}
gcc -c fonksiyon.c
gcc -c main.c
gcc main.o fonksiyon.o -o dikdortgen
./dikdortgen

Soru 4 — C# veya Java: local / global variable

Local (yerel) değişken: Bir metodun içinde tanımlanır; sadece o blokta görünür ve metod bitince ömrü biter.

Global (sınıf alanı / field): Sınıf seviyesinde tanımlanır; metodlar tarafından erişilebilir ve nesne yaşadığı sürece (veya static ise program boyunca) var olur.

using System;

class Ornek
{
    // "global" gibi düşünebilirsin: sınıf alanı (field)
    static int globalSayi = 100;

    static void Yazdir()
    {
        int localSayi = 5; // local: sadece bu metod içinde
        Console.WriteLine(localSayi);
        Console.WriteLine(globalSayi);
    }

    static void Main()
    {
        Yazdir();
        // Console.WriteLine(localSayi); // HATA: localSayi burada yok
        Console.WriteLine(globalSayi);   // OK
    }
}

Python notu: Python'da “global” kelimesiyle fonksiyon dışındaki değişkene yazmak için global anahtar sözcüğü gerekir.

Soru 5 — C++ referans iletimi göster

İstenen: Referans ile parametre geçirince fonksiyon, çağıranın değişkenini değiştirir.

#include <iostream>
using namespace std;

void kareYap(int &x) { // referans
    x = x * x;
}

int main() {
    int a = 6;
    kareYap(a);
    cout << a << "\n"; // 36
}

Bonus — “çıktı” sorularında kritik nokta (yan etki + değerlendirme sırası)

Hocanın verdiği şu tarz sorulara dikkat:

int fun(int *k) {
    *k += 4;
    return 3 * (*k) - 1;
}

int main() {
    int i = 10, sum1;
    sum1 = (i / 2) + fun(&i);
    printf("%d\n", sum1);
}
Neden tricky? C'de ifadelerin içindeki operandların değerlendirme sırası (sol mu önce, sağ mı önce) çoğu operatörde belirtilmemiştir. Bu örnekte i hem okunuyor (i/2) hem de fun(&i) içinde değiştiriliyor; arada “sequence point” olmadığı için sonuç derleyiciye göre değişebilir (teknik olarak tanımsız davranış).

Sınav pratiği: Hoca genelde “soldan sağa” varsayımıyla bekler. Ama açıklamada mutlaka “C'de evaluation order garanti değil” notunu yazmak seni güçlendirir.
